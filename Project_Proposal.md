## 2. 项目背景与动机

### 2.1 问题陈述

在日常计算机使用中，用户面临以下困扰：

1. **文件分散存储**：图片、音频、视频、文档等多媒体文件散布在各个目录中，难以统一管理和检索
2. **媒体播放不便**：需要打开不同的应用程序来播放不同类型的媒体文件
3. **数据备份繁琐**：手动复制粘贴文件夹效率低下，容易遗漏更新，且无法追踪备份历史
4. **存储空间不透明**：不清楚各类文件占用空间的比例，难以针对性地清理

### 2.2 解决思路

MediaVault 提供一个集成化的解决方案：
- **统一浏览**：通过树形目录+文件列表的双栏布局，快速浏览和搜索所有文件
- **内置播放**：集成媒体播放功能，支持音频/视频文件的直接播放
- **智能同步**：基于增量同步算法，仅复制变化的文件，显著提升备份效率
- **可视化统计**：通过饼图和柱状图直观展示文件分布，辅助存储管理决策

### 2.3 选题理由

1. **技术覆盖全面**：本项目涉及C# WinForms的所有核心技术栈，包括文件I/O、多线程、数据库、GDI+绘图、UI控件等
2. **实用性强**：文件管理是每个计算机用户的刚需，项目具有实际使用价值
3. **复杂度适中**：功能模块划分清晰，适合个人在一个开发周期内完成
4. **扩展性好**：可方便地添加新功能（如网络同步、云存储集成、标签系统等）

## 3. 系统架构设计

### 3.1 总体架构

```
┌─────────────────────────────────────────────────────┐
│                    表示层 (Presentation)              │
│  MainForm (TabControl)                              │
│  ├── 文件浏览器 Tab                                   │
│  ├── 媒体播放器 Tab                                   │
│  ├── 文件同步备份 Tab                                 │
│  ├── 统计图表 Tab                                     │
│  └── 数据库管理 Tab                                   │
├─────────────────────────────────────────────────────┤
│                    业务逻辑层 (Services)               │
│  ├── FileScannerService   (多线程文件扫描)            │
│  ├── FileSyncService      (异步增量同步)              │
│  ├── StatisticsService    (统计计算)                  │
│  └── DatabaseService      (数据持久化)                │
├─────────────────────────────────────────────────────┤
│                    数据模型层 (Models)                 │
│  ├── MediaFileInfo        (文件信息实体)               │
│  └── SyncJob              (同步作业实体)               │
├─────────────────────────────────────────────────────┤
│                    基础设施层                          │
│  ├── SQLite Database      (嵌入式数据库)               │
│  ├── System.IO            (文件系统操作)                │
│  ├── System.Drawing       (GDI+ 图形绘制)              │
│  └── System.Threading     (多线程支持)                  │
└─────────────────────────────────────────────────────┘
```

### 3.2 核心模块设计

#### 文件浏览器模块
- **目录树 (TreeView)**：延迟加载子目录，避免一次性加载整个文件系统
- **文件列表 (ListView)**：详细信息视图，展示文件名、大小、类型、修改日期
- **搜索过滤**：支持基于文件名的实时搜索
- **右键菜单**：复制路径、打开文件

#### 媒体播放器模块
- **播放列表 (ListBox)**：添加/移除/清空媒体文件
- **媒体播放**：调用系统默认播放器（Windows Media Player）播放音视频
- **格式支持**：MP3, WAV, WMA, MP4, AVI, WMV, MKV, FLAC, AAC

#### 文件同步备份模块
- **增量同步**：比较源文件和目标文件的最后修改时间，仅复制变化的文件
- **异步执行**：使用Task.Run + async/await，UI线程不受阻塞
- **进度报告**：IProgress<T> 模式实现的跨线程进度报告
- **取消支持**：CancellationToken 协作式取消

#### 统计图表模块
- **饼图 (Pie Chart)**：文件类型分布可视化（GDI+ FillPie）
- **柱状图 (Bar Chart)**：文件数量对比（LinearGradientBrush 渐变填充）
- **统计表格**：DataGridView 展示详细统计数据

#### 数据库管理模块
- **文件信息表**：存储扫描结果，支持增删查
- **同步历史表**：记录每次同步操作的详细日志
- **CSV导出**：将文件列表导出为CSV格式

## 4. 技术选型说明

| 技术点 | 选型方案 | 选型理由 |
|--------|---------|---------|
| UI框架 | WinForms | 课程教学框架，开发效率高，控件丰富 |
| 数据库 | SQLite | 轻量级嵌入式数据库，零配置，适合桌面应用 |
| 多线程 | Task + async/await | 现代C#异步编程最佳实践，避免Thread直接操作 |
| 图形绘制 | GDI+ | .NET原生图形库，无需第三方依赖 |
| Excel处理 | CSV格式 | 通用格式，无需第三方库 |
| 媒体播放 | Windows Media Player COM | 系统自带组件，支持格式广泛 |
| 单元测试 | MSTest | Visual Studio内置测试框架 |
